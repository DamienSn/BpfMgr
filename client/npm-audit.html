critical Prototype Pollution in immer
  • Module: immer
  • Published: November 18th 2021
  • Reported by:
  • CWE-843
  • CVE-2021-23436
  • Vulnerable: <9.0.6
  • Patched: >=9.0.6
  • Exploitability:

Overview

This affects the package immer before 9.0.6. A type confusion vulnerability can lead to a bypass of CVE-2020-28477 when the user-provided keys used in the path parameter are arrays. In particular, this bypass is possible because the condition (p === "proto" || p === "constructor") in applyPatches_ returns false if p is ['proto'] (or ['constructor']). The === operator (strict equality operator) returns false if the operands have different type.

Findings

  • react-scripts>react-dev-utils>immer

Remediation

Upgrade to version 9.0.6 or later

References

  • https://nvd.nist.gov/vuln/detail/CVE-2021-23436

  • Patched: >=1.0.0

Overview

Impact

The forge.debug API had a potential prototype pollution issue if called with untrusted input. The API was only used for internal debug purposes in a safe way and never documented or advertised. It is suspected that uses of this API, if any exist, would likely not have used untrusted inputs in a vulnerable way.

Patches

The forge.debug API and related functions were removed in 1.0.0.

Workarounds

Don't use the forge.debug API directly or indirectly with untrusted input.

Overview

The regex used for the forge.util.parseUrl API would not properly parse certain inputs resulting in a parsed data structure that could lead to undesired behavior.
